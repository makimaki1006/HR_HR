## 3. 重要な発見

### 3.1 R2 国勢調査基本集計の構造的制約

R2 国勢調査 人口等基本集計 (toukei=00200521) の **オンライン dbview に登録されている** 表では、以下のパターンが存在する:

| パターン | area | 0-14 含 | 例 |
|----|----|:----:|----|
| 全市区町村 (1,965) × 5歳階級 × 0-14 含 | ❌ 存在しない | - | - |
| 全市区町村 (1,965) × 5歳階級 × 15+ のみ | ✅ | ❌ | **0003445236** (A9 推奨) |
| 全市区町村 (1,965) × 各歳 (世帯人員ベース) | ❌ 存在しない | - | - |
| DID (1,038) × 5歳階級 × 0-14 含 × 国籍 | ✅ | ✅ | 0003445141 |
| 大都市 (359) × 5歳階級 × 0-14 含 × 国籍 | ✅ | ✅ | 0003445244 |
| 全国 (1) × 各歳 | ✅ | ✅ | 0003448228 (人口推計) |

### 3.2 結論: A9 候補が最良

- 0-14 を含む 1,965 市区町村テーブルは **dbview API 経由で取得不可** (恐らく「人口等基本集計 表 2-4」相当だが、市区町村レベルでは API 公開されていない可能性大)
- 0-14 が必要な場合は別データソース (人口推計 都道府県のみ、または CSV 一括ダウンロード で R2 表 2-4 ファイル直接取得) に切り替えるべき
- 本タスク範囲 (e-Stat API 経由取得) では `sid=0003445236` の **15+ のみ採用** が最適
