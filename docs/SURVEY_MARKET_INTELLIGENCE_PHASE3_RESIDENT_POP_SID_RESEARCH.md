## 4. 採用候補の詳細: `sid=0003445236`

### 表メタ

| 項目 | 値 |
|------|-----|
| 表名 | 男女，年齢（5歳階級），配偶関係，国籍総数か日本人別人口（15歳以上）－全国，都道府県，市区町村 |

### 軸構成

| 軸 | 名称 | 値（抜粋） |
|----|------|---------|
| cat01 | 表章事項 | 人口【人】 |
| cat02 | 国籍総数か日本人 | 国籍総数, うち日本人 |
| cat03 | 男女 | 総数, 男, 女 |
| cat04 | 配偶関係 | 総数, 未婚, 有配偶, 死別, 離別, 不詳 |
| cat05 | 年齢 | 15-19歳 〜 100歳以上 (5歳階級) + 再掲5項目 |
| cat06 | 地域 | 全国 + 47都道府県 + 全市区町村 + 政令市区 |

### area カバレッジ (推定)

WebFetch で「全国、都道府県、政令指定都市の区単位、および全市区町村」と確認。area_count=1965 は本要件 (designated_ward 175 件含む 1965) と一致。

**4 designated_ward サンプル存在確認**: WebFetch ベースの直接確認は dbview の area class 一覧を全件展開できないため未実施。**推測**: area_count=1965 は人口等基本集計シリーズ共通の「全国 + 47県 + 1917市区町村相当」と一致するため、`01101` 札幌市中央区 / `14131` 川崎市川崎区 / `27141` 堺市堺区 / `40131` 福岡市博多区 はすべて含まれる蓋然性が高い。**実測は metadata-only fetch (Step 5) で確定する**。

### データ取得設計

求人需給分析で純粋に「市区町村×男女×年齢5歳階級の人口」が欲しい場合、API パラメータで以下を固定:

- `cdCat02=000` (国籍総数)
- `cdCat04=000` (配偶関係総数)
- 残り cat03 (男女) × cat05 (年齢5歳) × cat06 (地域=1965) を全展開

これで純粋な男女×年齢5歳階級×市区町村人口表が得られる。
