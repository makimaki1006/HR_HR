## 既知の残課題 (今回修正対象外)

### R1. B2 「サービス業 (他に分類されないもの)」30.4% の解釈
- マッピング後も 30.4% を占める理由: HW industry_raw の **派遣業 / 建物管理 / ビルメンテナンス / 警備 / 清掃** はそもそも国勢調査の「サービス業 (他に分類されないもの)」大分類に分類されるため、マッピングは正しい
- 実態として新宿区の HW 求人の 30% がこの大分類なのは妥当 (派遣業界が新宿区集中の傾向)
- ❗ ただし「他に分類されないもの」というラベルが大きく出ると視覚的に「マッピング失敗の残骸」と誤認される懸念
- 提案: 別パスとして「派遣・人材」「建物管理サービス」などの **HW 独自カテゴリ** を表示する切替を検討

### R2. Design v2 KPI grid と legacy KPI grid の二重表示
- Web 表示では両方が見える
- 仕様上「印刷時は legacy が CSS で非表示」(Design v2 リファクタの遺物)
- 印刷プレビュー時の表示は OK だが、Web では冗長
- 提案: legacy を Web でも `display:none` にする (テスト互換性確認後)

### R3. /report/survey エンドポイントの体感速度
- 修正後でも初回ロードは数秒かかる (CR-9 関連 fetch を /report/survey 内で同期実行)
- 提案: postings (prefecture, municipality, industry_raw) 複合インデックス追加で大幅高速化可能
