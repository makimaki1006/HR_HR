# 媒体分析レポート拡張 指標定義

作成日: 2026-05-03

対象: ハローワーク分析システムV2 / 媒体分析レポート / 採用マーケットインテリジェンス

## 1. 指標設計の原則

この機能は、高精度な応募数予測ではなく、説明可能な採用判断を目的にする。

表示区分:

| 区分 | 意味 | 例 |
|---|---|---|
| 実測 | 統計または媒体データから直接集計できる値 | 対象職業人口、媒体求人数、通勤OD |
| クロス分析 | 実測同士を組み合わせた比較値 | 求人密度、人口あたり競合求人 |
| 推定 | 仮定を置いて算出する値 | 転職潜在母集団、推定流入、配信優先度 |
| 参考 | 粒度や定義に限界がある補助値 | 生活コスト補正、家賃proxy |

レポートでは、各指標に必ず上記の区分を表示する。

## 2. 配信優先度スコア

### 2.1 基本式

```text
配信優先度 =
  対象職業人口指数
+ 近接職種人口指数
+ 通勤到達性指数
+ 生活コスト補正後給与魅力度
- 競合求人密度指数
- 通勤負担指数
```

初期実装では、各要素を0〜100に正規化したうえで合成する。
ペナルティ項も「悪いほど高い0〜100指数」として保持し、減衰率に変換して加点側を割り引く。

推奨初期重み:

| 要素 | 重み | 備考 |
|---|---:|---|
| 対象職業人口指数 | 30 | 加点 |
| 近接職種人口指数 | 20 | 加点 |
| 通勤到達性指数 | 25 | 加点 |
| 生活コスト補正後給与魅力度 | 25 | 加点 |
| 競合求人密度指数 | 20 | 減衰率寄与 |
| 通勤負担指数 | 10 | 減衰率寄与 |

`penalty_reduction_pct` は **0〜30 の減衰率** で、競合求人密度と通勤負担によって `positive_score` を最大 30% 減衰させる。
名称が `_pct` で終わるのは、加点側 0〜100 指数とは異なる「パーセント減衰率」スケールであることを明示するため。

最終スコア:

```text
positive_score =
  (target_population_index * 30
 + adjacent_population_index * 20
 + commute_reach_index * 25
 + effective_wage_index * 25) / 100

penalty_reduction_pct =
  (job_competition_density_index * 20
 + commute_burden_index * 10) / 100

raw_score =
  positive_score * (1 - penalty_reduction_pct / 100)

distribution_priority_score = clamp(raw_score, 0, 100)
```

設計意図:

- `positive_score` が0点の地域は、ペナルティに関わらず必ず0点になる（人材が居ない地域に下駄を履かせない）。
- `penalty_reduction_pct` が満点(=30)の地域は、加点を 70% に減衰させる（競合激戦・通勤過酷地域を最大 30% 割り引く）。
- ペナルティは加点側に対する乗算減衰なので、positive が高くても競合や通勤負担が強ければ減衰する。
- 「人材豊富 × 競合激戦」が満点にならず、「人材ゼロ × 競合ゼロ」も基準点を得ない。

検算例:

| ケース | positive_score | penalty_reduction_pct | raw_score | 最終スコア |
|---|---:|---:|---:|---:|
| 全加点満点 / 減衰率0 | 100 | 0 | 100 × (1 − 0/100) = 100 | **100** |
| 全加点0 / 減衰率0 | 0 | 0 | 0 × (1 − 0/100) = 0 | **0** |
| 全加点満点 / 減衰率満点 | 100 | 30 | 100 × (1 − 30/100) = 70 | **70** |
| 全加点0 / 減衰率満点 | 0 | 30 | 0 × (1 − 30/100) = 0 | **0** |
| positive=70 / 減衰率=15 | 70 | 15 | 70 × (1 − 15/100) = 59.5 | **約60** |
| positive=80 / 減衰率=10 | 80 | 10 | 80 × (1 − 10/100) = 72 | **72** |

### 2.2 表示ルール

- 80〜100: 重点配信
- 65〜79: 拡張候補
- 50〜64: 維持/検証
- 0〜49: 優先度低

注意:

- 配信優先度は「採用しやすさの断定」ではない。
- 「検証すべき配信地域の優先順位」として表現する。

## 3. 対象職業人口

区分: 実測

定義:

```text
対象職業人口 = municipality_occupation_population
  where basis = 'resident'
  and occupation_code in target_occupation_codes
```

用途:

- その地域に対象職種の人がどれだけ住んでいるかを見る。
- 市区町村別の採用母集団の土台にする。

初期対象職業例:

- 輸送・機械運転
- 運搬・清掃・包装等
- 生産工程
- 建設・採掘

## 4. 近接職種人口

区分: 推定

定義:

```text
近接職種人口 =
  sum(occupation_population * transfer_weight)
  where occupation_code not in target_occupation_codes
```

初期重み例:

| 転換元職種 | 配送求人への重み |
|---|---:|
| 輸送・機械運転 | 1.00 |
| 運搬・清掃・包装等 | 0.80 |
| 生産工程 | 0.60 |
| 建設・採掘 | 0.45 |
| 販売 | 0.35 |
| サービス | 0.30 |

注意:

- 未経験歓迎求人では近接職種を含める。
- 近接職種人口は対象職業人口を除外して計算し、同一人口の二重計上を防ぐ。
- 職種別の転換重みは配送求人向けの初期値であり、製造・建築など別職種では個別に重みテーブルを定義する。
- 職種転換可能性は実測ではないため `推定` ラベルを付ける。

## 5. 競合求人密度

区分: クロス分析

定義:

```text
競合求人密度 = 競合求人件数 / max(対象職業人口 + 近接職種人口, 1)
```

表示:

- 人口1,000人あたり競合求人数に換算して表示する。
- 分母は定義式と同じく `対象職業人口 + 近接職種人口` に統一する。

```text
競合求人密度_1000 = 競合求人件数 / max(対象職業人口 + 近接職種人口, 1) * 1000
```

補助表示として、同一職種経験者だけの競争圧を見たい場合に限り、以下を別指標として併記できる。

```text
対象職業限定_競合求人密度_1000 = 競合求人件数 / max(対象職業人口, 1) * 1000
```

ただし、配信優先度スコアには `競合求人密度_1000` を使う。

用途:

- 人は多いが競合も多い地域を避ける。
- 人がいて競合が薄い地域を見つける。

## 6. 通勤到達性

区分: 推定

定義:

```text
通勤到達性 =
  normalized(
    destinationへのflow_share,
    destinationへのflow_count,
    origin側対象人口,
    昼夜間人口比
  )
```

初期実装:

- `commute_flow_summary` の目的地別TOP20を使う。
- TOP20に入る流入元は到達性を高く評価する。
- 市区町村間の詳細な年齢性別職業別ODは断定しない。

注意:

- 「A市からB市へ通う30代女性の特定職業人数」とは表示しない。
- 通勤ODと居住地側構成を組み合わせた推定として扱う。

## 7. 生活コスト補正後給与魅力度

区分: 参考

定義:

```text
生活コスト補正後給与魅力度 =
  媒体給与中央値
- 単身向け相当家賃proxy
- 物価補正
- 通勤負担
+ 通勤手当/車通勤メリット
```

表示方針:

- 金額そのものではなく、初期は0〜100の指数で表示する。
- 家賃は `1R`、`1K`、`1LDK` と断定しない。
- `単身向け相当`、`小世帯向け相当` と表現する。

データ:

- 住宅・土地統計: 家賃proxyの主データ。
- 小売物価統計: 月次補正。
- 家計調査/既存 `v2_external_household_spending`: 生活コスト補助。
- 地価/既存 `v2_external_land_price`: 地域コスト補助。

## 8. 給与競争力

区分: クロス分析

比較軸:

1. 媒体内競争力
2. HW/Indeedなど求人媒体上の相場
3. 賃金構造基本統計の企業規模別水準
4. 生活コスト補正後の見え方

企業規模区分:

| 表示 | 統計区分 |
|---|---|
| 大手水準 | 1,000人以上 |
| 中堅水準 | 100〜999人 |
| 中小水準 | 10〜99人 |
| 全体水準 | 企業規模計 |

注意:

- 10人未満の零細企業実態までは拾い切れない。
- ただし採用レポート上の比較軸としては十分有用。

## 9. 保守 / 標準 / 強気 母集団推定

区分: 推定

基本式:

```text
推定母集団 =
  常住地ベース職業人口
* 年齢性別適合率
* 通勤到達率
* 職種転換可能性
* 潜在転職率
* 競合補正
```

初期シナリオ:

| シナリオ | 潜在転職率の初期値 | 用途 |
|---|---:|---|
| 保守 | 1% | 既存経験者・近接地域中心 |
| 標準 | 3% | 通勤圏と近接職種を含む標準見積 |
| 強気 | 5% | 未経験歓迎・外縁部配信を広げた見積 |

制約:

- `保守 <= 標準 <= 強気` を必ず満たす。
- 応募者数ではなく、配信対象として現実的に狙える母集団と表現する。

## 10. レポートへの対応

| レポート表示 | 使用指標 |
|---|---|
| 結論サマリーカード | 配信優先度、母集団、競合、給与、生活コスト |
| 配信地域ランキング | 配信優先度、対象人口、通勤到達性、競合求人密度 |
| 採用余地マップ | 配信優先度 |
| 人材供給ヒートマップ | 対象職業人口、近接職種人口 |
| 給与競争力散布図 | 競合求人密度、生活コスト補正後給与魅力度 |
| 生活コストウォーターフォール | 媒体給与、家賃proxy、物価、通勤負担 |
| 年齢性別ピラミッド | 年齢性別人口 |
| 通勤流入Sankey | `commute_flow_summary` |
| 媒体運用アクション | 地域別KPI、配信優先度、訴求軸 |

## 11. 欠損時ルール

| 欠損データ | 表示 |
|---|---|
| 職業人口なし | 母集団推定を非表示。地域ランキングは参考表示に落とす。 |
| 通勤ODなし | 通勤到達性を参考扱いにし、昼夜間人口で代替。 |
| 家賃proxyなし | 生活コスト補正を非表示。額面給与比較のみ表示。 |
| 競合求人なし | 競合求人密度を非表示。対象人口中心のランキングにする。 |
| 市区町村コード不明 | 該当地域を集計対象から除外し、除外件数を表示。 |
