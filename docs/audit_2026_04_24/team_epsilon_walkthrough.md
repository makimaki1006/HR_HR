## ペルソナ A: 採用コンサル（顧客向け提案資料を作る）

**目標**: 顧客（介護・医療・小売の法人）に渡す「このエリア×この業種の採用戦略」資料を作成。具体性のある「穴場 / 激戦」マッピングと、自信を持って提示できる So What が必要。

### シナリオ A-1: 介護業界 × 東京で穴場エリアを探す

**想定操作フロー**:
1. ログイン (`templates/login_inline.html`)
2. ヘッダーフィルタで「東京都」選択 → 産業ツリーで「医療,福祉」 or 「老人福祉・介護」
3. 「採用診断」タブ (`/tab/recruitment_diag`) → 業種「老人福祉・介護」/ 雇用形態「正社員」/ 都道府県「東京都」/ 市区町村空
4. 「診断実行」→ Panel 7 (穴場マップ) で市区町村別の難度を確認
5. 「地図」タブ (`/tab/jobmap`) でビジュアル確認
6. 「企業検索」タブ (`/tab/company`) で具体企業をリスト化

**観察された詰まり / 迷い**:
- `templates/tabs/recruitment_diag.html:78-83` 市区町村は「都道府県を選択した後にカスケードロード」だが、Panel 7（穴場マップ）が市区町村未指定で動作するか UI 上明示されていない
- Panel 1 採用難度スコアの閾値（`recruitment_diag/handlers.rs:243-311` `classify_difficulty()`）が「1 万人あたり 3.0/7.0/15.0」と固定経験則。コメントに「経験則的な閾値（あくまで傾向。因果ではない）」とあり、**業種別にチューニングされていない**（介護と IT で同じ閾値）のはコンサル目線で弱い
- Panel 3 流入元分析の警告 `recruitment_diag/handlers.rs:23` `INFLOW_DATA_WARNING = "v2_flow_fromto_city は Turso 書き込み制限により約83%のみ投入済み"` → **顧客提出資料に「83%」は致命的**
- `tabs/jobmap.html:153` ヒートマップが `prefcode=13` 固定デフォルト。東京以外を見たい時に手動入力が必要であることに気付きにくい

**達成度: 3 / 5**
- ✅ 市区町村レベルの採用難度ランキングは取れる（Panel 7）
- ✅ 競合企業リストと Insight 示唆は取れる（Panel 4, 8）
- ❌ **資料化のためにスクリーンショットを 8 枚撮って Excel/PowerPoint に貼る必要がある**。Panel 1-8 を 1 枚 PDF にまとめる機能が無い
- ❌ Panel 3 の「83% 部分データ」警告が顧客提出に向かない

### シナリオ A-2: Indeed CSV をアップロードして HW と比較

**想定操作フロー**:
1. 「媒体分析」タブ (`/tab/survey`、`survey/handlers.rs:17-19`)
2. CSV をアップロード
3. `parse_csv_bytes_with_hints()` で自動判定 or 手動指定（`source_type`: indeed/jobbox/other/auto、`survey/handlers.rs:88-90`）
4. 「HW データと統合分析」ボタン押下（`survey/render.rs:273-277`）
5. 印刷用レポート表示 → PDF 化

**観察された詰まり**:
- アップロード上限 20MB（`lib.rs:39` `UPLOAD_BODY_LIMIT_BYTES`）。Indeed の大量 CSV では到達リスク
- CSV パースのヒントが「auto / indeed / jobbox / other」の 4 種類のみ。リクナビ・マイナビ・doda のテンプレ未対応で「other」フォールバック
- 統合分析ボタン後、`#survey-integration-result` に結果が swap されるが、**「印刷用レポートでは統合済みデータが含まれているのか / 別レポートか」が UI 上不明確**
- 「HTML ダウンロード」と「印刷用レポート表示」が並列に並ぶ（`survey/render.rs:278-287`）が違いが微妙

**達成度: 3 / 5**
- ✅ アップロード→分析→印刷レポートまで 1 タブ内で完結
- ✅ Indeed/JobBox の自動判定がある
- ❌ HW 統合の結果が「別ボタンを押さないと現れない」UX で、初見ユーザーは見落とす
- ❌ 民間サイトの自動データ取得は無く、毎回 CSV 手動アップロード

**達成度: 2 / 5**
- ✅ Insight タブ単独なら Excel/JSON/印刷 HTML が出せる

## ペルソナ C: 採用市場リサーチャー

## Cross-Persona 共通課題

### CC-1: 主要機能のナビ動線が悪い（最重要）

| 機能 | アクセス方法 | 問題 |
|---|---|---|
| 総合診断 (`/tab/insight`) | 詳細分析タブ → グループ切替 | 上位ナビに無い (`dashboard_inline.html:70-89`) |
| トレンド (`/tab/trend`) | 詳細分析タブ → グループ切替 | 同上 |
| ガイド (`/tab/guide`) | ヘッダー右上の「?」アイコン | アイコンのみで気付きにくい (`dashboard_inline.html:61-62`) |
| `/my/profile`, `/my/activity` | ヘッダー右上の「履歴」「設定」 | テキストリンクが小さい |
| 管理者ページ | URL 直打ち | 一般ユーザーには問題ないが UX 上でも非表示 |

**改善案**: 上位ナビに「総合診断」「トレンド」を追加（あるいは「分析」をドロップダウン化）。

### CC-2: HW 限定性の警告がノイズと化す

`HW_SCOPE_NOTE` / `CAUSATION_NOTE` (`recruitment_diag` モジュール) と Panel 5 バナー、フッター注意書き（`recruitment_diag.html:230-234`）が**同じ画面で 3 回繰り返される**。誠実だが、繰り返しすぎでユーザーは読まなくなる（学習性無視）。`MEMORY/feedback_hw_data_scope.md` の趣旨は守りつつ、**1 画面 1 回までに集約**する案を推奨。

### CC-3: エクスポート出力の統合 PDF が無い

| タブ | 出力 |
|---|---|
| 総合診断 | Excel xlsx + JSON + 印刷 HTML (`insight/export.rs`, `insight/render.rs:184-198`) |
| 媒体分析 | 印刷 HTML + HTML ダウンロード (`survey/render.rs:268-321`) |
| 採用診断 | （無し） |
| 地域カルテ | 印刷 HTML (`region_karte.js:99-110`) |
| 詳細分析 | チャート PNG のみ (`static/js/export.js`) |
| その他 | 同上 |

→ 「採用診断 + 媒体分析 + 詳細分析を 1 つの PDF に」できない。**コンサル / 営業の決定打**になりうる差別化機能が欠如。

### CC-4: 産業フィルタの「自由記述」と「ツリー」が混在

- 条件診断タブの産業: 自由記述 (`diagnostic.rs:66` `<input type="text">`)
- 採用診断タブの業種: 13 分類固定セレクト (`recruitment_diag.html:36-52`, `render.rs:29-43`)
- ヘッダーフィルタの産業: 2 階層ツリー (`/api/industry_tree`, `lib.rs:303`)

**3 種類の産業選択 UI が混在**し、内部マスタ（`SessionFilters.job_types` / `industry_raws` / 「未分類」）との整合が取れているか UI 上不明。

### CC-5: フィルタ反映の不一致

- 採用診断 / 条件診断 / 媒体分析の各タブ内に**独自フィルタ（業種・雇用形態など）**を持つ
- ヘッダーフィルタとタブ内フィルタの優先順位（タブ内フィルタが優先する設計、`recruitment_diag/handlers.rs:121-130` の `if params.prefecture.is_empty() { filters.prefecture.clone() }`）が UI 上明示されない

→ ユーザーは「ヘッダーで東京を選んだのに、採用診断で何も選んでないと結果が出ない」みたいな状態に陥る。

### CC-6: 初回ロードの遅延

- 企業プロフィール: 30-100 秒 (`company/handlers.rs:62`)
- DB 解凍: 起動時に 297MB → 1.6GB (`lib.rs:1040-1092`)
- 採用診断 8 パネル並列ロード: スピナー期間が長い

**Render 無料プランの cold start** 問題と相まって、初回アクセス体験が悪い。

## ベンチマーク（他ツール比）

| ツール | 強み | 弱み | 本ツールとの比較 |
|---|---|---|---|
| **RESAS** (経産省/内閣官房) | 公的統計の網羅性、UI 完成度、無料 | 求人データ無し、リアルタイム性無し | 求人データ・採用視点では本ツールが圧勝。一方、人口動態・産業構造の俯瞰は RESAS が上 |

| **業界レポート（介護労働安定センター等）** | 信頼性の高い統計、定型レポート | 年次更新、PDF のみ | 本ツールはインタラクティブ性で勝つが、定型レポート PDF 出力は弱い |

## 機能の Keep / Add / Remove 提案

### Keep（維持すべき価値）
1. **Insight 22 パターンの定型示唆** (`insight/engine.rs`): So What を自動生成する仕組みは独自性が高い
2. **HW + SalesNow + Agoop の統合**: 採用視点 × 企業情報 × 人流のクロス分析は他に無い
3. **HW 限定性の透明性**: 過去事故防止のための注記は誠実。形を変えて維持
4. **市場診断レーダー** (`/tab/diagnostic`): 自社条件入力の即時フィードバックは UX が良い
5. **採用診断 8 パネル並列ロード**: 体感速度を確保する良い設計

### Add（追加すべき機能）
1. **統合 PDF レポート**: 「現在の選択地域 + 産業」で全タブを統合した 1 PDF。コンサル / 営業の決定打
2. **47 都道府県横断比較表**: 1 画面で全都道府県の主要 KPI を並べる
3. **時系列タブの独立化**: ナビバーに「トレンド」「総合診断」を昇格
4. **シミュレーション機能**: 条件診断で「賞与 2.0 → 2.5 にしたら充足度はどう変わるか」
5. **ヘッダーフィルタ vs タブ内フィルタの整合表示**: パンくずに「ヘッダー: 東京都 / タブ内: 神奈川県（タブ内優先）」と明示

### Remove（削除 / 統合すべき）
1. **「未分類」産業の選択肢** (`overview.rs:97-99`): UI 上の意味が不明
2. **条件診断の自由記述「産業」フィールド** (`diagnostic.rs:66`): 採用診断と統一して 13 分類固定セレクトに
3. **`/my/profile`, `/my/activity` のヘッダー上テキストリンク**: 重要度低、ユーザーメニュードロップダウンへ集約
4. **詳細分析タブのグループ切替（構造分析/トレンド/総合診断）**: 各サブグループを上位ナビに昇格させ、二重階層を解消
5. **HW 限定性警告の重複バナー**: 1 画面 1 回までに集約（CC-2）

## 最終結論

V2 HW Dashboard は「HW + SalesNow + Agoop + e-Stat の統合」という独自価値を持つが、**情報の量に UI の動線が追いついていない**。

- 採用コンサル目線（A）: 機能は揃っているが、**顧客に渡せる体裁の出力が弱く、提案資料 1 本にならない** → 統合 PDF が決定打
- HR 担当目線（B）: 条件診断 + 採用診断 + 企業検索の 3 タブ往復が必要 → 1 タブ統合か、フローガイドが必要
- リサーチャー目線（C）: 1 地域の深掘りは強いが、**横断比較が弱い** → 47 都道府県比較表が決定打

3 ペルソナとも「機能はあるのに使いにくい」という感想に収束する。バックエンドは強固なので、**フロントエンド動線改善の ROI が極めて高い**。
