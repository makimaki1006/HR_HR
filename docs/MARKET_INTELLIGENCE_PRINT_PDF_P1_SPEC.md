# MarketIntelligence 印刷 / PDF P1 仕様

最終更新: 2026-05-06
対象 variant: `MarketIntelligence` (`src/handlers/survey/report_html/market_intelligence.rs`)
位置づけ: P0 (`@media print` ベース実装) の上に積む P1 仕様。実装範囲は本書で明示する範囲に限定する。

---

## 1. 目的とスコープ

### 1.1 目的
- A4 縦印刷 / PDF 出力時に、本ページを「営業に持ち出して即読める紙資料」として成立させる。
- 画面 UI の見た目をそのまま紙に流すのではなく、読み順 / ページ配置 / 強調を紙向けに再設計する。

### 1.2 P1 の範囲
- 既存 `@media print` CSS (P0 で `MI_STYLE_BLOCK` 内に追加済み) の調整。
- 印刷時の表示 / 抑制セレクタ追加。
- セクションのページ分割ヒント (`break-inside`, `break-before`) の付与。
- 注釈ブロックの再配置定義。

### 1.3 P1 の範囲外 (やらない)
- グラフライブラリの差し替え / 追加導入。
- 新しい集計指標 / 新しい DB クエリ。
- レイアウト全面刷新。
- 画面側 (screen) UI の再設計。
- `target_count` / 推定人数等を紙で再表示する設計 (Hard NG。後述 §7)。

---

## 2. PDF 読み順 (8 ステップ)

紙で上から下へ読んだとき、営業担当者の意思決定に必要な順序で並べる。DOM 順序は画面と共通でよいが、印刷時の視覚優先度 (フォントサイズ / 余白 / ページ先頭への配置) でこの順序を担保する。

| # | セクション | 役割 | 紙での扱い |
|---|---|---|---|
| 1 | 結論 | この案件は「採れる / 採りにくい」のサマリ | 1 ページ目最上部、最大級見出し |
| 2 | 採用示唆 | 営業がそのまま口頭で言える 1〜3 行 | 結論直下、枠囲み |
| 3 | 配信優先度 | A / B / C など優先ランク | 結論ブロック内に並列表示 |
| 4 | 市内順位 / parent_rank | 「市内 X 位 / N 件中」 | 数字を主表示。子分類順位より親順位を優先 |
| 5 | 実測根拠 | HW 実測 (件数・賃金中央値・前年比など) | 2 ページ目以降の冒頭、表形式 |
| 6 | 推定 β の補足 | β 表記が付く参考値の取り扱い | 表内に「推定β」ラベルで併記、本文では人数換算しない |
| 7 | 生活コスト・補助指標 | 家賃 / 物価 / 生活コスト等 | 推定値が混じる場合は「参考」ラベル必須 |
| 8 | 注釈・データ出典 | 出典 / 集計期間 / 既知の制約 | 最終ページ末尾、小フォント |

---

## 3. ページ配置

### 3.1 1 ページ目 (Above the Fold = 表紙兼サマリ)
- レポートタイトル (variant 名 / 対象地域 / 職種 / 出力日)。
- 結論ブロック。
- 採用示唆ブロック (1〜3 行)。
- 配信優先度バッジ。
- 市内順位 (parent_rank) を強調。
- 1 ページ目の中で読み切れるように、子分類ランキングや詳細表は流し込まない。

### 3.2 2 ページ目以降 (根拠データ)
- 実測根拠表 (HW 実測): 件数 / 賃金中央値 / 前年比 / 求人増減。
- 推定β を含む参考値表 (β ラベル明示)。
- 生活コスト・補助指標 (参考ラベル明示)。
- 規模帯比較がある場合は、中立表現に従う (「劣位」「縮小」等の評価語を使わない。MEMORY: feedback_neutral_expression_for_targets)。

### 3.3 最終ページ
- 注釈 / データ出典 / 集計期間 / 既知の制約。
- 小フォント (例: 9pt) で 1 段にまとめる。

### 3.4 改ページヒント
- 1 ページ目末尾の結論〜採用示唆ブロックは `break-inside: avoid` を付け、ブロック途中で割れないようにする。
- 「実測根拠」見出しは `break-before: page` 相当で 2 ページ目先頭に来るよう誘導する (P0 の `@media print` 内 `.mi-evidence-section { break-before: page; }` 追加で対応)。
- 表組みは `break-inside: avoid` を行単位で付け、行が縦に割れないようにする。

## 4. 印刷時に表示すべき要素

| カテゴリ | 要素 | 備考 |
|---|---|---|
| ヘッダ | レポートタイトル | variant 名込み |
| ヘッダ | 対象地域 | 都道府県 + 市区町村 |
| ヘッダ | 職種 | HW 職種カテゴリ |
| ヘッダ | 出力日 | サーバ生成日 |
| サマリ | 配信優先度 | A/B/C 等のラベル |
| サマリ | 市内順位 | parent_rank を主表示 |
| サマリ | 3 ラベル | 「実測」「推定β」「参考」 |
| 根拠 | 主要根拠 3 点 | 件数 / 賃金中央値 / 前年比 など |
| フッタ | データ注釈 | 出典 / 期間 / 制約 |

---

## 5. 印刷時に抑制すべき要素

| カテゴリ | 要素 | 抑制方法 |
|---|---|---|
| 装飾 | 過剰な背景グラデーション / 影 | `box-shadow: none; background: #fff;` |
| UI | タブ / トグル / モーダル起動ボタン | `display: none;` |
| UI | コピー / 共有 / ダウンロードボタン | `display: none;` |
| UI | ホバー前提のツールチップトリガ | 紙では到達不能のため `display: none;` |
| UI | ローディング / スケルトン | `display: none;` |
| 装飾 | 大型バナー画像 (もしあれば) | `display: none;` |
| ナビ | グローバルヘッダ / フッタ / サイドバー | 既存共通 print CSS で抑制済み想定 |

セレクタは MarketIntelligence variant のスコープに限定し、他 variant に副作用を出さないため `.mi-*` 接頭辞配下に閉じる。

## 6. 完了条件 (Definition of Done)

P1 実装完了とみなす条件:

1. A4 縦で 1 ページ目に「結論 / 採用示唆 / 配信優先度 / 市内順位」が収まり、目視で読める。
2. セクション見出し (`h2` / `h3`) が紙面下端に孤立しない (見出し直後のブロックと同ページ)。
3. 重要カード (結論 / 採用示唆 / 配信優先度) がページ境界で割れない。
4. 推定β の値が、紙面上で「人数」「想定人数」「母集団」などと誤読される表現になっていない。`(推定β)` ラベルが値の直近に必ず表示されている。
5. parent_rank が子分類順位より視覚的に優先されている (フォントサイズ / 配置で担保)。
6. 注釈・出典が最終ページに集約され、本文と混ざっていない。

検証方法 (P1 完了確認時):
- Chromium 系ブラウザの「印刷プレビュー → 用紙: A4 / 向き: 縦 / 余白: 既定」で目視確認。
- 1 ページ目スクリーンショットを LLM 視覚レビュー (MEMORY: feedback_llm_visual_review) に通す。

## 7. 絶対遵守事項 (Hard NG)

P1 実装中、以下は仕様書 / コード / コメントいずれにも混入させない。

- `target_count` / `estimated_population` / 推定人数 / 想定人数 / 母集団人数 等、「人数として読める推定値」の表記。
  - 推定値は必ず `推定β` ラベル + 比率 / 指標として扱う。人数化しない。
- 秘密値 (token, .env 値, 接続文字列, API key) の本書 / コードへの転記。
- 新しいグラフライブラリの導入提案。
- variant の大規模改修 / 共通レイアウトの再設計提案。
- 評価語 (「劣位」「集中」「縮小」など、提案先地域を貶める表現) の紙面流入。

## 8. 実装メモ (P1 担当者向け)

- 編集対象: `MI_STYLE_BLOCK` 内 `@media print { ... }` ブロックのみで完結させる。Rust 側ロジック (`market_intelligence.rs` の HTML 生成) には触らない方針。
- セレクタは `.mi-` 接頭辞に限定。
- ページ分割: `break-before: page;` / `break-inside: avoid;` / `break-after: avoid;` を併用。
- 抑制クラス: 画面側で既に存在する `.mi-noprint` (なければ新設) に集約。新設する場合も CSS のみ。
- 注釈ブロックは `.mi-print-footnotes` のような専用クラスで最終ページに寄せる。実装は CSS の `break-before: page;` で十分。`position: fixed;` は使わない (印刷バグの原因)。
