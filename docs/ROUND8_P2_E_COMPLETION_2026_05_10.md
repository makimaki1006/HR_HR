#### 新 (P2-E、強調部分が追加)
```
X 軸: CSV 求人数。Y 軸: 国勢調査 R3 経済センサス全産業従業者数...
象限分割は対象自治体の中央値を基準。

【サンプル数について (本 CSV: N 集約点)】 ← 動的カウント
本図は CSV に含まれる対象自治体数に依存します。
実顧客 CSV では数十点以上のケースもあり、点数が増えると象限分割の中央値判定がより安定します。
サンプル少 (5 点未満) では中央値が偏りやすいため、各点の絶対値と下表の推奨アクションを
主判断材料とし、象限位置は補助的にご参照ください。

【東京都特別区部の規模注意】 (P1-7 既存)...
```

### 変更点

| 要素 | 変更 |
|---|---|
| `<strong>サンプル数について (本 CSV: N 集約点)</strong>` | 新規追加。N は `points.len()` で動的に展開 |
| 「本図は CSV 自治体数依存」 | 設計の明示 |
| 「実顧客 CSV では数十点以上」 | 過度に言い訳せずスケール感を伝える |
| 「サンプル少 (5 点未満) では中央値が偏りやすい」 | 5 点未満の場合の注意 |
| 「各点の絶対値と下表の推奨アクションを主判断材料」 | 表 + 推奨アクション (P2-A) との関係明示 |
| 「象限位置は補助的にご参照」 | 散布図を補助情報と位置づけ |
| `format!` 引数化 | 静的 string → 動的 (n=points.len()) |
| 「象限分割は 4 自治体...」 | 「象限分割は対象自治体...」に変更 (4 固定 → 一般化) |
