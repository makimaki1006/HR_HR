## 1. 検証目的と制約

### 1.1 ユーザー指示

municipality_occupation_population の推定モデルが「採用レポート商品品質」に耐えるかを判断するため、**精度評価プロトタイプ** を作成 (本実装ではない)。

### 1.2 ultrathinking の論点整理

#### 真の市区町村別職業正解データの存在

- e-Stat 国勢調査就業状態等基本集計には職業大分類があるが、**市区町村粒度での公開 API 提供は未確認** (前回 Worker C 調査結果)。
- → 真の正解データは取得できない。**絶対精度の検証は不可能**。

#### 検証指標の選択

絶対精度が測れない以上、検証は以下の組み合わせで代替する:

| 検証指標 | 内容 | 何を判定できるか |
|---------|------|----------------|
| 都道府県再集計誤差 | 各モデルで pref 別合計 vs 仮 ground truth | scaling で必ずゼロ化 (情報量小) |
| **モデル間 Spearman ランキング相関** | A vs B、B vs C' の市区町村順序の相関 | 補正項に対する順位の頑健性 |
| **代表地域の値比較** | 新宿区/川崎市/豊田市等の数値が直感に合うか | 定性的な妥当性 |
| 保守/標準/強気レンジ幅 | turnover_rate 1%/3%/5% で人数 | レンジが広すぎるか、人数表示できるか |
| **物流/製造/建設の説明力** | これらの職業で工業都市が上位に来るか | 商品価値の核心 |

### 1.3 ローカル DB の制約 (実装可能なモデル範囲)

| データ | ローカル | 本プロトで使用 |
|--------|:-------:|:-------------:|
| `v2_external_population` (市区町村人口) | ✅ 1,740 行 | F1 |
| `v2_external_population_pyramid` (年齢性別) | ✅ 15,660 行 | F2 |
| `v2_external_daytime_population` (昼夜間) | ✅ 1,740 行 | F4 (昼夜間補正) |
| `commute_flow_summary` (JIS 通勤 OD) | ✅ 27,879 行 | (簡易省略) |
| `municipality_code_master` (JIS) | ✅ 1,917 行 | 結合 |
| **`v2_external_industry_structure`** | ❌ **不在** | **F3 → 実装不可** |
| **`v2_salesnow_companies`** | ❌ **不在** | **F6 → 実装不可** |

→ プロトタイプは **Model A / B / C' (F1+F2+F4)** のみ。F3 と F6 はデータ不在で評価不能。

### 1.4 仮 Ground Truth

真の正解データがないため、以下の仮 ground truth を構築:

```
都道府県就業者数 ≈ 都道府県生産年齢人口 × 全国就業率 (75%)
都道府県職業人口 = 都道府県就業者数 × 全国職業構成比 (R2 国勢調査ベース)
```

**limitation**: この仮 ground truth は地域差を持たない (全国比率を都道府県人口で按分したもの)。本プロトの「絶対精度」は意味を持たない。検証の本質は **モデル間の相対比較 + 定性評価**。

## 2. 実装したモデル

### 2.1 Model A: 単純総人口比按分

```
muni_occ[muni, occ] = pref_occ_pop[pref(muni), occ] × (muni_total_pop / pref_total_pop)
```

最も単純。子供・高齢者を含むため、就業者推定としては精度が落ちる。

### 2.2 Model B: 生産年齢人口比按分

```
muni_occ[muni, occ] = pref_occ_pop[pref, occ] × (muni_age_15_64 / pref_age_15_64)
```

就業可能人口比に基づくため、A よりも妥当性が高い (理論上)。

### 2.3 Model C': Model B × F4 昼夜間補正

```
F4 = daytime_pop / nighttime_pop  (basis='workplace' のみ、0.1〜5.0 にクランプ)
muni_occ_raw[muni, occ] = Model B[muni, occ] × F4[muni]
muni_occ[muni, occ] = muni_occ_raw × scaling[pref, occ]    # 都道府県集計を pref_occ_pop に一致させる
```

workplace 基準: 新宿区 (day/night=2.25) → workplace 人口大。郊外住宅地 → workplace 人口小。

### 3.10 Model F: E3 + F6 SalesNow 残差補正 (Worker E, 2026-05-04 最終ラウンド)

#### 3.10.1 構成

```
raw[muni, occ] = baseline (F1+F2)
              × F3[muni, occ] ** 1.5             (E3 と同じ)
              × F4_occupation_weighted[muni, occ] (E2 由来)
              × F5[muni]                          (E と同じ)
              × F6_factor[muni, occ]              (新規、本社過剰減衰 + 工場ブースト)
```

設計原則 (Worker E 設計):

- **F6 と F3 は独立**: F3 (経済センサス産業構成) は「統計上の従業者数」、F6 (SalesNow 本社) は「企業実体」。両者の差分が**本社オフィスのみで実工場ゼロ**の状態を示す
- **F6 は乗算** (置換ではない): F6 が 1.0 のとき Model F は Model E3 と完全一致
- **二重計上回避**: F6 は「本社過剰時に現業職を減衰」「工場過剰時に現業職をブースト」のみ、絶対量を出さない

#### 3.10.2 SalesNow データソース

- **使用データ**: `data/salesnow_companies.csv` (467,262 行、46 カラム、UTF-8 エンコーディング、ローカル CSV、READ-only)
- **集約結果**: `data/generated/salesnow_aggregate_for_f6.csv` (11,071 セル、市区町村 × JSIC 大分類 × 従業員総数)
- **抽出カラム**: prefecture, address (→ 市区町村抽出 99.99% 成功), sn_industry (→ JSIC 大分類マッピング), employee_count
- **マッピング**: `SALESNOW_TO_JSIC` 辞書で SalesNow 44 業種 → JSIC 大分類 (D 建設業 / E 製造業 / H 運輸業 等)
- **Turso V2 経由は不使用**: ローカル CSV で完結、READ-only 制約遵守

#### 3.10.3 F6 計算式の最終形

```python
# 1. 本社過剰度 hq_excess_ratio
sn_intensity[muni, ind]    = sn_emp[muni, ind] / national_sn_emp[ind]
estat_intensity[muni, ind] = industry_emp[muni, ind] / national_emp[ind]
hq_excess_ratio = sn_intensity / estat_intensity

# 2. 工場過剰度 factory_excess_ratio
factory_excess_ratio = estat_intensity / sn_intensity

# 3. 産業別閾値 (本社減衰)
F6_HQ_EXCESS_THRESHOLD_BY_IND = {
    "D": 3.0,  # 建設業 (本社+現場混在を保護)
    "E": 4.0,  # 製造業 (大手メーカー本社=工場併設を保護)
    "H": 2.5,  # 運輸業 (純粋オフィス本社の典型)
}

# 4. 本社減衰 (alpha=0.6)
if hq_excess_ratio > 閾値:
    damping = 1 / (1 + 0.6 × (hq_excess_ratio - 1.0))
    f6_factor[occ] *= 1 - blue_collar_share[ind, occ] × (1 - damping)

# 5. 工場ブースト (beta=0.6, threshold 1.5, cap 1.4)
if factory_excess_ratio > 1.5:
    boost = min(1.0 + 0.6 × (factory_excess_ratio - 1.0), 1.4)
    f6_factor[occ] *= 1 + blue_collar_share[ind, occ] × (boost - 1.0)
```


## 4. なぜ Model A/B/C' で職業差が出ないか (理論的説明)

### 4.1 数式上の証明

Model A:
```
muni_occ[muni, occ] = pref_occ[pref, occ] × (muni_pop / pref_pop)
                    = K[occ] × pref_employment[pref] × ratio[muni, pref]
```

ここで `K[occ] = NATIONAL_OCCUPATION_RATIO[occ]` は職業ごとの全国比率。

→ 同じ都道府県内の市区町村の職業 occ の値は、**`ratio[muni, pref]` に比例**。
→ 都道府県を越えた比較も、`pref_employment[pref] × ratio[muni, pref] = muni_pop × employment_rate × K[occ]`。
→ **全職業の市区町村順位は muni_pop 順** (定数 K[occ] が乗じられるだけ)。

Model B も同様 (muni_pop が age_15_64 に置換されるだけ)。

Model C' は F4 (昼夜間) で workplace 補正するが、**全職業に同係数**なので順位は変わらない (新宿区が全職業で底上げされるだけ)。

### 4.2 結論

**職業ごとの地域差を出すには「職業に応じて係数が変わる補正項」が必須**。これが F3 (産業構成) の役割。

例: 製造業比が高い豊田市
```
F3[豊田, 生産工程] = 1.5    (生産工程は底上げ)
F3[豊田, 専門技術] = 0.7    (専門技術は減少)
```

これにより、豊田の生産工程が「人口順 6 位」から「全国 TOP 5」に上昇する可能性がある。
