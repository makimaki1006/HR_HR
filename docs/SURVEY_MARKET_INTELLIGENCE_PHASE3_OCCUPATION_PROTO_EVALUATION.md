## 1. 検証目的と制約

### 1.1 ユーザー指示

municipality_occupation_population の推定モデルが「採用レポート商品品質」に耐えるかを判断するため、**精度評価プロトタイプ** を作成 (本実装ではない)。

### 1.2 ultrathinking の論点整理

#### 真の市区町村別職業正解データの存在

- e-Stat 国勢調査就業状態等基本集計には職業大分類があるが、**市区町村粒度での公開 API 提供は未確認** (前回 Worker C 調査結果)。
- → 真の正解データは取得できない。**絶対精度の検証は不可能**。

#### 検証指標の選択

絶対精度が測れない以上、検証は以下の組み合わせで代替する:

| 検証指標 | 内容 | 何を判定できるか |
|---------|------|----------------|
| 都道府県再集計誤差 | 各モデルで pref 別合計 vs 仮 ground truth | scaling で必ずゼロ化 (情報量小) |
| **モデル間 Spearman ランキング相関** | A vs B、B vs C' の市区町村順序の相関 | 補正項に対する順位の頑健性 |
| **代表地域の値比較** | 新宿区/川崎市/豊田市等の数値が直感に合うか | 定性的な妥当性 |
| 保守/標準/強気レンジ幅 | turnover_rate 1%/3%/5% で人数 | レンジが広すぎるか、人数表示できるか |
| **物流/製造/建設の説明力** | これらの職業で工業都市が上位に来るか | 商品価値の核心 |

### 1.3 ローカル DB の制約 (実装可能なモデル範囲)

| データ | ローカル | 本プロトで使用 |
|--------|:-------:|:-------------:|

→ プロトタイプは **Model A / B / C' (F1+F2+F4)** のみ。F3 と F6 はデータ不在で評価不能。

### 1.4 仮 Ground Truth

真の正解データがないため、以下の仮 ground truth を構築:

```
都道府県就業者数 ≈ 都道府県生産年齢人口 × 全国就業率 (75%)
都道府県職業人口 = 都道府県就業者数 × 全国職業構成比 (R2 国勢調査ベース)
```

**limitation**: この仮 ground truth は地域差を持たない (全国比率を都道府県人口で按分したもの)。本プロトの「絶対精度」は意味を持たない。検証の本質は **モデル間の相対比較 + 定性評価**。

## 2. 実装したモデル

### 2.1 Model A: 単純総人口比按分

```
muni_occ[muni, occ] = pref_occ_pop[pref(muni), occ] × (muni_total_pop / pref_total_pop)
```

最も単純。子供・高齢者を含むため、就業者推定としては精度が落ちる。

### 2.2 Model B: 生産年齢人口比按分

```
muni_occ[muni, occ] = pref_occ_pop[pref, occ] × (muni_age_15_64 / pref_age_15_64)
```

就業可能人口比に基づくため、A よりも妥当性が高い (理論上)。

### 2.3 Model C': Model B × F4 昼夜間補正

```
F4 = daytime_pop / nighttime_pop  (basis='workplace' のみ、0.1〜5.0 にクランプ)
muni_occ_raw[muni, occ] = Model B[muni, occ] × F4[muni]
muni_occ[muni, occ] = muni_occ_raw × scaling[pref, occ]    # 都道府県集計を pref_occ_pop に一致させる
```

workplace 基準: 新宿区 (day/night=2.25) → workplace 人口大。郊外住宅地 → workplace 人口小。

### 3.5 保守/標準/強気レンジ

```
turnover 1%/3%/5% を base に乗算 → agg/cons = 5.0 (固定)
```

→ レンジ幅は turnover_rate の単純倍率なので**設計上固定**。base 推定値の精度に完全依存。

### 3.6 Model E: F1+F2+F3+F4+F5+F6 (2026-05-04 追加実装)

#### 3.6.1 構成

```
baseline = Model B (F1 + F2)
F3[muni, occ] = (Σ_i industry_share[muni, i] × weight[i, occ])
              / (Σ_i national_share[i] × weight[i, occ])
F4[muni]      = daytime_pop / nighttime_pop  (workplace 基準、0.1〜5.0 クランプ)
F5[muni]      = 1 + (inflow_pop / nighttime_pop) × 0.3  (0.5〜2.0 クランプ)
F6            = 1.0  (SalesNow 不在のため stub)
raw[muni, occ] = baseline × F3 × F4 × F5 × F6
muni_occ_E[muni, occ] = raw × scaling[pref, occ]   (都道府県集計を pref_occ_pop に一致)
```

## 4. なぜ Model A/B/C' で職業差が出ないか (理論的説明)

### 4.1 数式上の証明

Model A:
```
muni_occ[muni, occ] = pref_occ[pref, occ] × (muni_pop / pref_pop)
                    = K[occ] × pref_employment[pref] × ratio[muni, pref]
```

ここで `K[occ] = NATIONAL_OCCUPATION_RATIO[occ]` は職業ごとの全国比率。

→ 同じ都道府県内の市区町村の職業 occ の値は、**`ratio[muni, pref]` に比例**。
→ 都道府県を越えた比較も、`pref_employment[pref] × ratio[muni, pref] = muni_pop × employment_rate × K[occ]`。
→ **全職業の市区町村順位は muni_pop 順** (定数 K[occ] が乗じられるだけ)。

Model B も同様 (muni_pop が age_15_64 に置換されるだけ)。

Model C' は F4 (昼夜間) で workplace 補正するが、**全職業に同係数**なので順位は変わらない (新宿区が全職業で底上げされるだけ)。

### 4.2 結論

**職業ごとの地域差を出すには「職業に応じて係数が変わる補正項」が必須**。これが F3 (産業構成) の役割。

例: 製造業比が高い豊田市
```
F3[豊田, 生産工程] = 1.5    (生産工程は底上げ)
F3[豊田, 専門技術] = 0.7    (専門技術は減少)
```

これにより、豊田の生産工程が「人口順 6 位」から「全国 TOP 5」に上昇する可能性がある。

### 4.4 Model E 投入後の実証結果 (2026-05-04 追加)

#### 4.4.1 仮説の部分検証

豊田市の生産工程 Model A=23,367 → Model E=38,560 (x1.65) と F3 効果は出た。
しかし TOP 10 には到達せず 20 位どまり。

#### 4.4.2 想定外の問題: F4 (昼夜間) が都心オフィス街を過度に upweight

| 地域 | day/night | F4 値 | F4 が下流に与える影響 |
|------|---:|---:|------|
| 東京都 港区 | 〜4.0 | 4.0 | 全職業 x4 の底上げ → 製造業 TOP 10 入り |
| 東京都 中央区 | 〜3.5 | 3.5 | 全職業 x3.5 → 製造業 TOP 6 |
| 東京都 新宿区 | 〜2.4 | 2.4 | 全職業 x2.4 → 製造業 TOP 11 |

F3 で港区の製造業比は低いはずだが、scaling で都道府県集計を pref_occ_pop に一致させる際、
**F4 の高い区が全体の `raw` 合計を吊り上げ**、相対的に F3 の効果が薄まる現象が発生。

#### 4.4.3 改善余地 (F6 投入時の方向性)

| 案 | 説明 | 期待効果 |
|----|------|---------|
| F4 を職業別に強度差を付ける | オフィス職 (管理/専門技術/事務/販売) には F4 強、現業職 (生産工程/輸送機械/建設/運搬) には F4 弱 | 都心区が現業職 TOP 10 から消える |
| F3 を強める (べき乗 1.5) | F3^1.5 で産業構成補正の効果を増幅 | 工業都市が現業職で浮上 |
| F6 で SalesNow 残差補正 (alpha=0.25) | 各市区町村の従業員数比率で残差調整 | 中規模工業都市の精度向上 |

## 8. ユーザー判定基準への直接回答

### 8.1 ユーザー判定基準

> 1. 市区町村別の人数を断定できるほど安定しないなら、人数表示はやめる
> 2. ランキングが安定するなら『配信優先度』『ターゲット厚み指数』として使える
> 3. レンジが広すぎるなら、人数ではなく『見込み濃淡』
> 4. 物流/製造/建設で説明力が出るなら、商品価値は残る

### 8.2 直接回答

| 基準 | 回答 |
|-----|------|
| 1. 人数を断定できるか | **NO**、F3 投入後でも真の正解データなしのため断定不可。**人数表示はやめるべき** |
| 2. ランキングが安定するか | **モデル間では安定 (相関 0.997)、しかし全職業同順位 = 情報量ゼロ**。F3 投入後に職業別ランキングが分散するかが分かれ目 |
| 3. レンジが広いか | turnover_rate の単純倍率 (5×) 固定。広くはないが、**base 推定が不安定**なら結局 cons/agg もブレる |
| 4. **物流/製造/建設で説明力が出るか** | **現状: NO**。F3 投入後: **要再検証**。ここが商品価値の最後の砦 |
