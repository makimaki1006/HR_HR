## 8. 監査結果総評

- **データ鮮度**: 主要統計の出典・年次は概ね明示。実測/推定/参考のラベル運用も導入済み。
- **同一値整合**: 集計定義の異なる「中央値/平均月給」が同名で複数並ぶ表示揺れが最大の弱点 (P1-1, P1-2)。
- **PDF 間整合**: 4 PDF は別 session で対象が異なるため直接比較は限定的だが、PDF2 で「対象県の誤入力」がシステム側で検知されないまま生成された事故が確認 (P0-1)。
- **ドメイン不変条件**: 数式上の構成比合計 100% は許容範囲内、47 県上限も問題なし。一方、給与単位 (時給/月給/年俸) の取り扱いで P0 級違反が 2 件 (P0-2, P0-3)。

## 9. 次ラウンド推奨

1. **P0-1 県名と市区町村名の整合チェック**: 入力検証で「市区町村名から都道府県を逆引きし、ユーザー入力県と一致しない場合は警告 / 拒否」する仕組み (現状ではラベル整合エラーが本文に伝播)。
2. **P0-2/P0-3 給与単位の事前判定**: 月給/時給/年俸の混在 CSV を検出し、ネイティブ単位以外を集計から除外するロジックを最低賃金比較・産業別集計に拡張。`feedback_unit_consistency_audit.md` の横展開。
3. **P0-4 職業×地域マトリクスの年齢階級ラベル付与**: 現状 `管理的職業従事者 従業地ベース (実測) 0 人` が 22 行並ぶ → `管理的職業従事者 従業地ベース 15-19 歳 (実測) 0 人` のように年齢階級カラム追加。
4. **P1-1/P1-2 中央値・平均ラベルに集計定義タグ付与**: 「給与中央値 (全体 IQR 除外後)」「給与中央値 (正社員ネイティブ単位)」など読者が定義差を即把握できる接頭/接尾辞付与。
5. **P1-3 売上単位統一**: 「百万円」「億円」「円」の混在を解消。10 億円以上は「億円」、1 億円未満は「百万円」、それ以下は表外・「-」推奨。
6. **逆証明 (`feedback_reverse_proof_tests.md`) の追加**: 「平均月給下限 ≤ 平均月給 ≤ 平均月給上限」「IQR 法での外れ値除外件数 ≤ 全体の 10% 」「都道府県別件数合計 = サンプル件数」等の不変条件をビルド時にテスト化。
