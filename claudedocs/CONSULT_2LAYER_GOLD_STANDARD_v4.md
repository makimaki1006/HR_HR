# 商談準備レポート 複合診断 ゴールド基準 v4（到達目標）

このファイルは「商談準備レポートの複合考察がどの品質・構造に到達すべきか」の見本(ゴールド)。
上位LLM(Opus)が **evidence_pack のみを入力**(＝アプリのGeminiと同条件)に作成した。
実装(コミット 4a24f92)と実 Gemini 出力は、この構造・精度規律を再現できているかで評価する。

対象データ例: 大分県大分市の求人CSV + 公的統計 + 企業DB（顧客の提示給与 25〜30万円・採用3名を入力）。

---

## 設計原則（この4つが「で?結局何?」を消す）
1. **2層**: 単品層（観測ひとつずつに「だから」の着地）＋ 複合層（単品を織った戦略示唆）。両方必要。
2. **全着地**: すべての示唆が「観測 → **だから（具体的な結論）**」で終わる。宙に浮く観測をゼロに。
3. **正確性監査**: 各「だから」がデータの限界（粒度・観測可能性・未発火シグナル）を超えていないか。
4. **取りこぼしゼロ**: 全 evidence を「単品で使う / 複合で使う / 除外（理由付き）」のどれかに決着させる。

---

## §0 リード仮説（この1行だけ持ち帰る）
給与は市場の上位1割（E-008、確認済み）。**だから"給与を上げる"は打ち手にしにくい。ただし休日など他の条件は未確認 — そこが弱いと条件も論点に残る。** 商談は ①「応募は来ているか」＋②「御社の年間休日は市場上位に入るか」から入る。

> 精度ポイント: "条件で勝てている"と書いてはいけない。確認済みは**給与のみ**（客の提示給与という入力から算出）。休日など他条件は未確認。「給与では」と限定する。

## §2 複合の示唆（単品を織って戦略に = 統合の層。3〜4本）

**複合1｜給与では勝てているが休日は未確認。だから"条件で勝てているか"は休日次第。**（E-008＋E-023/022）
→ 面談で御社の休日を確認。市場上位に入れば給与×休日の二軸／入らなければ休日が"弄れる条件レバー"として残る。どちらでも打ち手が定まる。

**複合2｜地元は細るが、大分市は"働きに来る安い街"。だから母集団は広域＋移住で取り、強い給与を当てる。**（E-002/021/003＋E-017/012/013/020＋E-008）
→ 最大レバー: 配信圏に別府・由布・臼杵が入っているか確認。家賃が安いので移住採用も現実的。

**複合3｜市場は厳しすぎない＋勝者不在。だから結果は市場でなく運用（配信・見せ方）で決まる。**（E-001/005＋E-006/007/016）
→ 「市場が厳しいから採れない」とは言えない。分散市場は可視性ゲーム。§0の"問題は条件でない"を市場側から裏付ける。

**複合4｜顕在転職者は伸びにくいが、廃業・欠員で人は出る。だから狙う層を"動いている人"から広げる。**（E-003＋E-018/019＋E-011は呼び水）
→ 受け身層・再就職層に届く設計（継続施策）。顕在層だけの奪い合いを避ける。

## §3 限界（この資料が言えないこと）
給与は客の提示値入力／休日は"カードで観測できた分"／通勤・失業・求人倍率は県・全産業計／選考・応募数は市場データに無い（面談で埋める）／個社は各1社観測。

## 精度監査で確定した「言ってはいけないこと」（未発火シグナル由来）
実装のガードはこれらを **pack の fired 状態から動的に生成**する（大分の値をハードコードしない）:
- 給与が低い（S-02未発火）／給与が最賃に近い（S-04未発火、底は堅い）
- 転職層が「薄い」「厚い」（S-08/09いずれも未発火 → **全国並み**）
- 家賃が「高い」（S-22未発火 → **安い**）
- タグが「少ない/横並び」（S-26未発火）／人気が「集中」（S-27未発火）
- 正社員以外が「多い」（S-30未発火 → 正社員中心）
- 休日について「市場に無い」（観測限界 → "カードで見えにくい"に留める）
- 個社（E-011/027）を示唆・優先の主役にする（→呼び水/除外どまり）
